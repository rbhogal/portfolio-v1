Stanza (Client work)

An admin tool for the Stanza mobile app.

Client required a CRUD web app admin tool for multiple admin/moderators that consumes two RESTful APIs (Spotify and Genius APIs). Created a secure login page using bcrypt, dashboard connecting to their GraphQL backend database displaying data in data tables using Material UI components, and multiple tools to create, add, delete, and edit content for their mobile app.

Developed alongside the backend dev and UX designer as the sole frontend developer, provided and implemented solutions for the frontend given the requirements, and provided direction on an aesthetically pleasing UI, all coded with future scalability in mind.

React / Redux / NPM / Spotify API / Genius API / Material UI

*Site and repo are private
*Images blurred per client's request

Dashboard Analytics

The dashboard allows admins to view and keep track of their content and user data stored in their database. The data is fetched from the GraphQL backend and displayed in data tables using Material UI components. Card components are applied to a grid layout providing easy scalability for the future.

Create and Add Data

Admin/mods can search either a track, album, or playlist provided by the Spotify API. Results are displayed in cards. Selecting an item leads to another page where content is fetched using the Genius API and displayed in a data table. Pagination allows you to cycle through songs in an album or playlist. Admin/mods can then select the data they want using checkboxes, assign their tags, and add it to their database. Three sections were requested for the page. One to select content, another to preview the selected content, and lastly a section to view content already stored in their database.

Challenges

The biggest challenge was handling asynchronous data.

I ran into an issue where, upon refresh, state in a useState hook was being updated in more places than one (at the same time), causing a bug in my code when sending data to the database. Thankfully due to my last project I had done (Game Save), I knew it must be an async issue. After some time I was able to identify and rectify the bug by making sure state was only updated in one place.

Learning Material-UI

With no UI design for the app I was given free reign on my choice of visual library or CSS to use, and the design direction I would like to take. I decided to learn Material UI specifically for this project as I felt it would save time, and I could worry less on how it looks and focus more on the javascript. However I did end up adding some custom css for a cleaner look.

I learned it in three hours with NetNinja's tutorial series. And anytime I got stuck I read the documentation and figured it out as I went along.
